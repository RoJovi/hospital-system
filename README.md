# 医院管理系统

## 项目简介

这是一个基于C语言开发的控制台版医院管理系统，实现了患者管理、医生管理、药品管理、挂号管理、住院管理、费用管理等核心功能。系统通过链表管理数据，并采用哈希表优化查找性能，支持管理员和普通用户两种角色登录。

---

## 目录结构

```
code/

├── cost.c/h          # 费用管理
├── hospitalization.c/h # 住院管理
├── data.c/h          # 数据持久化（文件读写）

├── recommend.c/h     # 智能药品推荐（规则引擎）

```

## 数据结构

| 结构体            | 说明                                                         |
| ----------------- | ------------------------------------------------------------ |
| `Patient`         | 患者信息（ID、姓名、年龄、性别、电话、入院日期、诊断、治疗方案、总费用） |
| `Doctor`          | 医生信息（ID、姓名、年龄、性别、科室、专业领域、电话、排班） |
| `Medicine`        | 药品信息（ID、名称、规格、厂家、价格、库存、生产日期、有效期） |
| `Registration`    | 挂号记录（单号、患者ID、医生ID、挂号日期、科室、费用、状态） |
| `Cost`            | 费用记录（单号、患者ID、日期、项目、金额、支付状态）         |
| `Hospitalization` | 住院记录（住院号、患者ID、医生ID、入院日期、床位、诊断、总费用、状态） |

## 核心算法

### 1. 哈希表查找（O(1)）
- 患者、医生、药品、费用均使用哈希表存储
- 查找操作从链表 O(n) 优化为 O(1)
- 采用拉链法解决哈希冲突

### 2. 优先队列（挂号排队）
- 按优先级排序：急诊 > 老年人 > 普通
- 支持入队、出队、查看队首操作

### 3. 规则引擎（药品推荐）
- 基于关键词匹配的症状-药品映射
- 无需API调用，完全本地运行

## 系统特色

1. **无需网络**：所有功能本地运行
2. **数据持久化**：自动保存，重启不丢失
3. **高性能查找**：哈希表 O(1) 复杂度
4. **统一界面风格**：所有菜单使用相同边框样式
5. **优先级排队**：急诊、老年人优先叫号
6. **智能推荐**
